# XHS Pilot — 分阶段执行指南

> **本目录包含 6 个独立的阶段执行文件**，每个文件是一个完整闭环的开发任务。
> 执行时按顺序逐个进行，**每完成一个阶段验收通过后再进入下一个**。

## 执行原则

1. **一次只做一个阶段** — 不要把多个阶段合并执行
2. **严格边界** — 每个阶段文件有「禁止事项」，必须遵守
3. **闭环验证** — 每个阶段都有验收检查清单，全部通过才算完成
4. **参考主文档** — 每个阶段标注了需要参考的 `implementation_plan.md` 章节

## 阶段总览

| 阶段 | 文件 | 内容 | 预估 |
|------|------|------|------|
| 1 | `phase-1-scaffold.md` | 项目脚手架 + 数据库 + Docker | 1 天 |
| 2 | `phase-2-sample-analysis.md` | 样本录入 + LLM 分析管线 | 2 天 |
| 3 | `phase-3-retrieval.md` | 检索层（结构化 + 向量 + 降级） | 1 天 |
| 4 | `phase-4-creation-studio.md` | 创作工作台（核心交互页） | 2-3 天 |
| 5 | `phase-5-ui-pages.md` | Dashboard + 样本库 + 详情 + 画像 | 2 天 |
| 6 | `phase-6-productization.md` | 产品化收尾（本地存储/备份/PWA/文档） | 1 天 |

## 依赖关系

```
Phase 1 → Phase 2 → Phase 3 → Phase 4
                                  ↓
                              Phase 5 → Phase 6
```

## 如何使用

给 AI 编码 Agent 时：
1. **必须先给它 `CONVENTIONS.md`**（全局约束，每次都带）
2. 再给它**当前阶段的文件**
3. 必要时补充 `implementation_plan.md` 的指定章节
4. 明确说：**"只完成这个阶段的任务，不要提前实现后续阶段的功能"**
5. 完成后按验收清单逐项检查
