# OPT-007：高密度研究工作台重设计

> **目标**：将现有前端从“页面分散、卡片偏多、工作面偏松”的界面重构为更符合 XHS Pilot 产品本质的“高密度研究工作台”。这轮优化重点解决信息过于零散、首屏过大、交互链路不够紧凑的问题，让收集、分析、归类、创作四个环节更像同一张案头工作面。

## 前序依赖

Phase 6 已完成，当前主线已经具备 Dashboard、样本库、样本详情、创作工作台、风格画像、离线页与对应 API / SSE 交互能力。

本轮属于新一轮前端重设计。执行时以前端当前功能边界为准，但**不继承 `OPT-006` 的视觉骨架与版式假设**；如 `OPT-006` 与本轮方向冲突，以 `OPT-007` 为准。

## 具体任务

### 0. 本轮产品表达与交互定位

- 整体产品表达统一为：**内容研究台 + 档案库 + 参考式创作工作面**
- 不再沿用“通用 SaaS 后台”或“首页报表 + 多个大页面”的默认表达。
- 不把产品做成灵感白板、自由拖拽画布或社媒平台运营后台；主交互应围绕：
  - 搜索与筛选
  - 扫读与比对
  - 引用与跳转
  - 局部展开与局部修正
  - 带参考的创作输出

### 1. 全局壳层与导航骨架重构

- 将当前“左侧大导航 + 主内容区”的主骨架重构为更紧凑的工作台布局：
  - 桌面端以**顶部工具条 + 中央主工作区 + 按需出现的上下文侧栏/抽屉**为主
  - 移动端以**顶部栏 + 抽屉/底部动作入口**为主
- 顶部工具条至少承载以下能力：
  - 产品标识
  - 核心导航入口
  - 当前页面标题或工作上下文
  - 常驻主动作入口：录入样本、开始创作
- 页面切换后，外壳应尽可能稳定，减少每个页面都像“新场景”的割裂感。
- 不改变既有路由结构与导航语义，只重做布局层级与展示方式。

### 2. 全局视觉系统与排版系统重构

- 全局视觉语言改为“研究档案感 + 编辑型秩序 + 克制品牌红”：
  - 背景使用暖白、纸灰、浅雾灰中性色
  - 文字以高对比深墨色为主
  - 小红书语义红仅作为少量强调色，不做大面积铺陈
- 视觉层级应优先依赖以下手段，而不是大卡片堆叠：
  - 分隔线
  - 面板边界
  - 密度差异
  - 字重与字号层级
  - 局部底色与轻阴影
- 排版系统改为中文优先的高可读组合：
  - 正文、表单、元数据优先长期阅读与高频操作
  - 标题允许有更强的编辑感，但不做夸张杂志风
  - 优先本地 / 系统可用字栈，不引入新的 UI 框架或依赖型字体方案
- 收紧间距体系，建立更明确的紧凑密度规则，避免首屏大片留白与空洞大块。

### 3. Dashboard 重构为“资产指挥台”

- 首页不再强调报表感或大篇幅欢迎区，而是改成一屏内可扫读的资产指挥台：
  - 资产快照条
  - 最近录入 / 最近生成
  - 待处理与异常状态
  - 热门参考与高价值样本
- 所有关键区块应服务于两个问题：
  - 系统现在沉淀了什么
  - 我下一步最值得做什么
- 图表表达保持轻量，不新增图表库；优先列表、排行、分布条、时间线、微型统计组件。

### 4. 样本库重构为“研究档案台”

- 样本库从卡片浏览页改为高密度的研究档案台：
  - 顶部固定检索条
  - 筛选区可折叠但默认可达
  - 结果区以行式 / 表格式 / 档案式列表为主，不以卡片瀑布为主
- 每条结果优先呈现：
  - 标题
  - 缩略图或封面提示
  - 标签 / 类型 / 赛道
  - 状态
  - 被引用价值
  - 更新时间
- 结果项应支持更短路径的查看与处理：
  - 行内展开摘要或关键分析信息
  - 或侧边预览抽屉
  - 但不新增独立路由
- 保留既有回收站、录入抽屉、分页、状态刷新、软删除与恢复语义。

### 5. 样本详情重构为“样本 dossier”

- 样本详情不再是一组松散的大区块，而应更像一份可检索、可索引的档案：
  - 顶部摘要区
  - 区块索引 / 吸顶 section nav
  - 原始内容
  - 解析结果
  - 认知结果
  - 关系网络
  - 人工修正
- 详情页应优先支持两类行为：
  - 从上到下系统阅读
  - 直接跳到某个分析区快速核对
- 关系网络与引用记录必须更可跳转、更可扫读，避免埋在长页面深处。
- 人工修正区要明显是“操作区”，与阅读信息区分开。

### 6. 创作工作台重构为“流程轨道 + 稿纸面板”

- 创作工作台不再使用固定三等分大栏位作为主要视觉结构。
- 新骨架改为：
  - 顶部任务输入条 / 输入 sheet
  - 中央主流程轨道：任务理解、参考检索、策略形成、生成进度
  - 右侧或底部可停靠的稿纸面板：承载结构化输出与最终成稿
- 中央区域要成为真正的“工作过程视图”，而不是多个同等权重的大盒子并排摆放。
- 结果面板要更像稿纸、编辑稿或最终输出抽屉，支持在流程中随时查看，而不是把右栏做成另一个页面。
- 必须保留现有 SSE 流式事件协议、状态机与客户端职责边界；本轮只改表现层结构与交互组织。
- 去掉阶段性研发文案和“Phase 4”残留表达，统一为正式产品文案。

### 7. 风格画像页重构为“风格索引与档案”

- 风格画像列表页改为风格索引页：
  - 信息密度高于当前收藏架 / 卡片感
  - 优先展示名称、样本数、核心标签、最近更新时间
- 风格画像详情页改为档案页：
  - 顶部是画像摘要
  - 中段是样本引用与标签结构
  - 编辑区为独立操作面板
- 保持“人工策展、人工分组”的产品边界，不扩展为自动推荐、自动归纳或新的 Agent 流程。

### 8. 离线页、空态与状态反馈统一收口

- 离线页必须并入新设计系统，不再像孤立的提示页。
- 全站空态、错误态、加载态、禁用态统一收口：
  - 信息要短
  - 层级要清楚
  - 不依赖颜色单独表达状态
- 在紧凑界面下仍需保证焦点态、错误反馈、键盘可达性与可读性。

### 9. 测试与验证

- 更新现有静态渲染测试，使其对新的壳层、主动作入口、页面文案与关键结构建立断言。
- 补充以下页面的空态 / 有数据态检查：
  - Dashboard
  - 样本库
  - 样本详情
  - 创作工作台
  - 风格画像
  - 离线页
- 手工验证至少以下断点：
  - `375`
  - `768`
  - `1024`
  - `1440`
- 确认在新布局下：
  - 无明显横向滚动
  - 顶部工具条与抽屉不互相遮挡
  - 主动作入口始终可达
  - `prefers-reduced-motion` 被尊重
- 实现完成后运行：
  - `npm run lint`
  - `npm test`
  - `npm run build`

## 禁止事项 ❌

- ❌ 不要修改任何后端 API、数据库 schema、SSE 事件协议或 Agent 输出契约
- ❌ 不要改变既有路由结构、页面数据依赖与 RSC / Client 边界
- ❌ 不要引入 Tailwind、组件库、图表库或其它前端框架
- ❌ 不要继续沿用“左侧大导航 + 大卡片 + 大留白”的旧骨架
- ❌ 不要把本轮做成自由拖拽画布、无限白板或重交互动效实验
- ❌ 不要扩展为深色模式改造
- ❌ 不要新增与当前产品边界无关的新功能，如命令面板、自动推荐、发布链路或协作体系
