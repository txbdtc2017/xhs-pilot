# OPT-006：编辑型内容工作台重设计

> **目标**：将现有前端从通用 SaaS 风格收口为更符合 XHS Pilot 产品语义的“编辑型内容工作台”，让样本沉淀、分析归类与再创作三条链路在同一套界面语言下更成熟、更清晰。

## 前序依赖

## 具体任务

### 1. 全局设计系统重构

- 将全局视觉语言调整为“纸感档案 + 克制品牌红”：
  - 背景与表面使用暖白、纸张、灰墨中性色
  - 主动作色保留克制红色，不做高饱和平台复刻
  - 状态色与辅助色降低饱和度，避免 SaaS 仪表盘感
- 标题、正文、表单建立新的排版层级：
  - 标题带编辑感和内容感
  - 正文与表单优先长期阅读和输入效率
- 继续使用现有 `variables.css` 与 `globals.css`，不引入 Tailwind、组件库或图表库。

### 2. 全局壳与导航重构

- 保留桌面端左侧导航，但改为更像工作台侧栏而不是菜单栏。
- 品牌区要更明确表达“内容资产与创作工作台”定位。
- 导航区下方增加两个常驻主动作入口：
  - 录入样本
  - 开始创作
- 移动端使用顶部栏 + 抽屉导航，避免内容区与导航互相挤压。
- 不改变现有路由结构、导航项与页面职责。

### 3. 页面层重设计

#### Dashboard

- 改成“资产总览页”，强调资产沉淀、最近活动、下一步动作。
- 核心数据卡、分布面板、最近活动、热门参考使用更像内容工作台的层级结构。
- 图表继续使用 CSS 原生轻量方案，不新增图表依赖。

#### 样本库

- 改成“内容档案库”视角：
  - 筛选区更稳定、更像检索面板
  - 结果区优先标题、标签、引用价值、状态信息
- 桌面端采用更高信息密度的档案式展示；移动端回落为易读卡片。
- 保留回收站、录入抽屉、分页、状态刷新、软删除语义。

#### 样本详情

- 调整为“样本档案”布局，强化五个区域之间的视觉边界：
  - 原始内容
  - 解析结果
  - 认知结果
  - 关系网络
  - 人工修正
- 原始内容优先阅读体验，分析结果优先扫描效率，关系网络优先跳转效率。

#### 创作工作台

- 保留三栏结构与现有 SSE 流式交互。
- 视觉上并入新设计系统：
  - 左栏为任务输入 sheet
  - 中栏为策略与检索时间线
  - 右栏为稿纸式结果区
- 去掉“Phase 4”阶段性文案，改为正式产品表达。
- 不修改状态机、流式事件协议、客户端职责边界。

#### 风格画像

- 列表页改为更像策展式风格集合。
- 详情页改为风格档案布局，样本区、标签区、编辑区边界更明确。
- 保持“人工策展”定位，不扩展为自动推荐或自动归纳产品。

#### 离线页

- 与主设计系统统一，表达“可查看工作台外壳，但核心在线能力不可用”的限制。

### 4. 测试与验证

- 更新现有静态渲染测试，确保新文案与关键结构可被断言。
- 补充关键页面的空态 / 有数据态断言：
  - Dashboard
  - 样本库
  - 样本详情
  - 风格画像
  - 离线页
- 实现完成后运行：
  - `npm run lint`
  - `npm test`
  - `npm run build`

## 禁止事项 ❌

- ❌ 不要修改任何后端 API、数据库 schema、Agent 输出契约
- ❌ 不要改变现有路由语义、页面数据依赖与 RSC / Client 边界
- ❌ 不要引入 Tailwind、组件库、图表库或其它前端框架
- ❌ 不要把本轮扩展为深色模式改造
- ❌ 不要把界面做成小红书平台后台的直接复刻
