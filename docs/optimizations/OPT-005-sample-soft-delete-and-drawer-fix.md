# OPT-005：样本软删除与录入抽屉遮挡修复

> **目标**：为样本库补齐可回收的删除能力，并修复录入抽屉与底层样本列表互相干扰的问题，让用户可以在不重做整页布局的前提下完成样本管理。

## 前序依赖

Phase 6 完成，且当前主线已经具备样本录入、样本详情、样本删除 API 基线与样本库页。

## 具体任务

### 1. 样本软删除模型

- 为 `samples` 表新增 `deleted_at` 字段，用于表达“已移入回收站”状态。
- 软删除只写入 `deleted_at`，不物理删除：
  - `sample_images`
  - `sample_analysis`
  - `sample_visual_analysis`
  - `sample_embeddings`
  - `task_references`
  - `style_profile_samples`
- 恢复样本时清空 `deleted_at`，保留原有分析、图片与引用关系。
- 彻底删除时才沿用当前物理删除逻辑与存储清理逻辑。

### 2. 样本查询与页面视图

- 样本列表新增 `view=active|trash` 视图参数，默认 `active`。
- `active` 视图仅展示 `deleted_at IS NULL` 的样本。
- `trash` 视图仅展示 `deleted_at IS NOT NULL` 的样本。
- 现有搜索、赛道、内容类型、封面风格、高价值、日期区间、分页在两种视图下都继续生效。
- 样本详情读取允许返回已软删除样本，供回收站中的恢复与彻底删除操作使用。

### 3. API 与操作语义

- 保留现有 `/api/samples/[id]` 主路由。
- `DELETE /api/samples/[id]` 语义改为：将样本移入回收站。
- 新增 `POST /api/samples/[id]/restore`：恢复已删除样本。
- 新增 `DELETE /api/samples/[id]/permanent`：彻底删除已在回收站中的样本，并清理本地存储对象。
- 对不存在样本、重复恢复、对活跃样本直接彻底删除等情况返回明确错误，不做静默容错。

### 4. 样本库与详情页 UI

- `/samples` 顶部增加“正常样本 / 回收站”切换。
- `active` 视图中：
  - 样本列表卡片提供“移入回收站”动作
  - 样本详情页头部提供“移入回收站”动作
- `trash` 视图中：
  - 样本列表卡片提供“恢复”与“彻底删除”动作
  - 样本详情页头部提供“恢复”与“彻底删除”动作
- 本轮不新增批量操作，不新增独立回收站页面，不做样本库整体信息架构重排。

### 5. 录入抽屉遮挡修复

- 保留当前右侧录入抽屉，不改为独立页面或页内展开表单。
- 抽屉打开时必须：
  - 锁定页面滚动
  - 阻止底层样本列表交互
  - 保证 backdrop 与 panel 始终覆盖在页面内容之上
- 小屏下继续使用全宽抽屉。
- 回收站视图不显示录入入口，避免在回收站里继续触发“新建样本”。
