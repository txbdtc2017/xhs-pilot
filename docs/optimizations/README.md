# XHS Pilot — 后续优化执行指南

> **本目录用于承载 Phase 1-6 完成之后的持续优化执行文档。**
> 每个 `OPT` 文件都是一份可直接交给 AI 或工程师执行的指导文档，写法与 `docs/phases/*.md` 保持一致。

## 这套目录是干什么的

- `docs/phases/`：系统从 0 到 1 的阶段建设文档
- `docs/optimizations/`：系统进入可用状态后的后续优化文档
- `docs/roadmap.md`：暂不执行、后续再做的路线项

## 使用原则

1. **多步骤优化先写文档** — 在实现前，先新增下一份 `OPT-xxx` 文档
2. **一轮优化一份文档** — 每个新优化主题都新建新编号，不复用旧 `OPT`
3. **文档是执行说明书** — 不记录状态、不写复盘、不承担 roadmap 作用
4. **边界要明确** — 每份文档必须包含目标、任务、禁止事项和验收清单

## 如何使用

给 AI 编码 Agent 时：

1. 先给它 `docs/CONVENTIONS.md`
2. 再给它 `docs/optimizations/CONVENTIONS.md`
3. 再给它当前目标 `OPT` 文档
4. 明确说：**“先按这份 OPT 文档执行，不要顺手扩展到其他优化主题”**
5. 完成后按文档中的验收清单逐项检查

## 何时新建下一份 OPT

出现以下情况时，应新建新的 `OPT-xxx` 文档，而不是直接实现：

- 涉及多个子系统或多个文件域
- 需要先统一约束、接口或边界
- 需要完整验收清单
- 属于后续持续优化，而不是单点小修

命名和写法规范见 [CONVENTIONS.md](./CONVENTIONS.md)。
