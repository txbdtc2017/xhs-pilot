# XHS Pilot — 演进规划（Phase 2 & Phase 3）

> 本文档仅记录未来阶段的高层规划。每个阶段开始前再做详细设计。
> Phase 1 的完整编码方案见 `implementation_plan.md`。

---

## Phase 2：风格能力升级（Phase 1 后 4~6 周）

> **目标**：从"单篇参考"升级到"风格体系参考"

### 新增功能

| 功能 | 说明 |
|------|------|
| 风格画像页完整版 | 系统自动归纳画像规则，不再只依赖人工总结 |
| 系统推荐"这篇也像这个风格" | 基于 Embedding 相似度推荐关联样本 |
| 多样本规律自动归纳 | Profile Agent 正式上线 |
| 样本相似度推荐 | 样本详情页展示"相似样本" |
| 高价值样本权重影响检索排序 | Rerank 阶段考虑 `is_high_value` 权重 |
| 生成结果版本管理 | 同一任务可多次生成，支持版本对比 |
| 创作时可指定风格画像 | 生成时选择特定画像作为风格基准 |
| **内容级样本防重** | Embedding 相似度 > 0.95 提示"疑似重复"，用户选择合并或保留 |
| **互动数据加权排序** | Rerank 阶段引入 engagement 信号（log 加权） |
| **独立 OCR 服务** | 拆分 GPT-4o Vision 端到端，引入低成本 OCR 降低分析费用 |
| **可视化数据导入/导出** | 设置页"一键备份/恢复"按钮，替代 CLI 脚本 |

### 验收标准

系统能形成风格画像，创作时能按画像风格生成，而不只是依赖单篇样本。

---

## Phase 3：反馈闭环（Phase 2 后 4~6 周）

> **目标**：系统具备"学习"能力，越用越精准

### 新增功能

| 功能 | 说明 |
|------|------|
| 发布数据回填 | 记录实际发布后的互动数据 |
| 用户采用记录分析 | 统计用户选了哪些标题/正文版本 |
| 检索权重自动调整 | 含互动数据混合打分公式 |
| 个人风格偏好建模 | 系统学习用户倾向的风格 |
| 完整复盘页面 | Feedback Agent 正式上线 |
| LLM 动态路由调度 | Agent 编排层升级为 LLM 判断任务类型→动态调度 |
| **样本指纹 Hash + 版本链管理** | SimHash 指纹去重 + 同文章多版本关联 |

### 验收标准

系统能回答"你适合什么风格"、"哪些标题对你更有效"。

---

## 成功判断标准

| 阶段 | 关键指标 | 达标线 |
|------|----------|--------|
| Phase 1 | 沉淀 → 分析 → 策略 → 生成 闭环跑通 | 分析结果结构化且可用于检索；生成结果有明确参考依据 |
| Phase 2 | 风格画像形成 | 至少 3 个风格画像，每个包含 ≥10 篇样本 |
| Phase 3 | 反馈闭环生效 | 系统推荐的风格与用户实际偏好匹配度 ≥70% |
